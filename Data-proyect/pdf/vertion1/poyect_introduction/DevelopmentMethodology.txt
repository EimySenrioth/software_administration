Development: 
The "AcBi" project is a multiplatform mobile application designed for researchers for the 
proactive detection of corruption. We will utilize an Agile methodology (Scrum). 
The system follows a Clean Architecture approach, organizing code into independent layers to facilitate 
maintenance.

Development Approach
It will follow a Hybrid Development model: React with JavaScript for interactive UIs like dashboards, 
timelines, and charts via vis.js and D3.js with Rust for logic bussiness and obfuscation, 
complemented by Serde and Reqwest for JSON and HTTP management.

I will incorporate Go and Python for concurrent logic in modules such as Process Tracking, and C++ for 
high-performance sections if required. The full lifecycle will be managed from initial design Tauri 2.0 Mobile, 
with React Native.

Comprehensive testing will be conducted using Jest and Cargo for unit tests for cryptography, in order to
ensure anonymity and counter-intelligence protection.

Architectural Design
The structure are Clean Architecture with Local-First. To maintain total control 
over sensitive data, the app runs without third central servers. 

The code is divided into clearly separated layers:

UI Layer (Frontend): React with vis.js and D3.js for the dashboard, visualizations, and dynamic diagrams.

Core Layer (Backend): Rust for entities and main modules, using Serde/Reqwest 
for API management, with support for Go, Python, or C++ in concurrent or performance-heavy subcomponents.

Infrastructure Layer = *Database: Encrypted local storage using SQLCipher.

Security: SHA family for hashing, wolfSSL for cryptography and VPN support, and custom Rust-based anti-forensics 
(sandbox and memory dump detection).

Obfuscation: Obfuscator.io for code protection.
